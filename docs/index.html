API Tester CLI Documentation

Overview

The API Tester CLI tool allows you to test API endpoints and generate API documentation. It supports various configurations and output formats.

Commands

1. test

Tests a specified API endpoint.

Usage:

apitester test --url <URL> --method <METHOD> [--body <BODY>] [--headers <HEADER>]

Options:

  • --url: API URL (required)
  • --method: HTTP method (default: GET)
  • --body: Request body (optional)
  • --headers: Request headers (optional, comma-separated)

2. generate-doc

Generates API documentation in JSON format.

Usage:

apitester generate-doc --output <OUTPUT_FILE>

Options:

  • --output: Output file path (default: ./apidoc.json)

Examples

Testing an API Endpoint

./apitester test --url "https://fakestoreapi.com/products" --method "GET"

Generating Documentation

./apitester generate-doc --output "apidoc.json"

Configuration

Configuration settings can also be specified in a YAML file:

url: "https://fakestoreapi.com/products"
method: "GET"
body: ""
headers:
  - "Content-Type: application/json"

Getting Help

Run apitester --help to see a list of all commands and options.
